Project Overview

Manpower Recruitment Firm (imaginary company) is a recruitment company which offers applicants to their client companies. The new data admin has noticed the company's out-of-date operations and introduced changes such as data cleaning and data visualization to utilize data in order to generate informed decisions for the company's benefit.
Leading the initiative, the new data admin is tasked to create a dashboard. With the given applicant data covering the last six months, the project aims to evaluate firm and recruiter performance based on the success rates of applicants with a focus on key metrics such as applicant experience levels, education, and regional effectiveness.

Data Cleaning

A. Handling Blank Spaces

  • Select all cells in the table (CTRL + A).
  • Navigate from Home > Find & Select > Go To Special > Blank Cells.
  • On the formula bar, enter the text NA.
  • Click CTRL + ENTER.

B. Adding The Field "Age Range"

This field is added to provide a range, which allows to easily group applicants based on age. With added interpretation per range, the data could be easily visualized.

Age Range
Verbal Interpretation
55+
Old
31 - 54
Middle Age
0 - 30
Adolescent
Here's the formula in Excel =IF([@Age] ≥ 55, "Old", IF([@Age] ≥ 31, "Middle Age", IF([@Age] < 31, "Adolescent", "Invalid"))) .

C. Adding The Field "Province/City"

Based on the objectives, regional effectiveness is one of the key metrics to determine success rates. The data admin discovered that the field Full Address contained messy data. With this, the data admin came up with the following solutions:
  • Use conditional formatting.
    • Here's a sample data from the field Full Address: Brgy 123 Zone 7 Hills Cainta Rizal
    Rizal is a province. Before populating the field Province/City, conditional formatting is used to highlight records based on a defined condition. For this case, records/rows containing the word Rizal is highlighted.
    Conditional Formatting > Highlight Cells Rules > Text that Contains... > Enter the province or city
  • Sort by color all the data from field Full Address before populating the field Province/City.

D. Adding The Field "Region Group"

  • Create a new table with two fields Province/City and Region Group. This table is named Place.
  • To populate the field Region Group, this formula is used =VLOOKUP([@[Province/City]], Place, 2, FALSE). This formula searches the look-up value [@[Province/City]] on the table Place and returns an exact value from the specified column 2 of the same row.

E. Adding The Field "Status"

The field Remarks is composed of messy data. With this, the data admin decided to add another field named Status. The five applicant statuses are Passed, Failed, New, Pending and Withdrawn.
Here's the formula in Excel: 
=IF([@Remarks]="Passed", "Passed", IF([@Remarks]="New Endorsement", "New", IF([@Remarks]="Ready for Interview", "Pending", IF([@Remarks]="Pending", "Pending", IF([@Remarks]="Withdrawn", "Withdrawn", IF([@Remarks]="Invalid", "Withdrawn", "Failed"))))))

F. Adding The Field "Experience Range"

To efficiently visualize the data, the field Experience Range is added. It provided a range and interpretation that helps in grouping applicants based on their years of work experience.
